## Вывод
В ходе работы команда освоила основные инструменты Git и GitHub для совместной разработки. Были выполнены операции клонирования репозитория, создания коммитов, работы с ветками, Pull Request, Push, Pull и Fetch. Также удалось на практике столкнуться с конфликтами слияния и научиться их разрешать. Работа показала, что использование веток, регулярный Pull и проверка изменений через Pull Request делают командную разработку более удобной и безопасной.
