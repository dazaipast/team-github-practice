## Вопросы
1. Репозиторий — это хранилище проекта в Git со всеми файлами и историей изменений.
2. Локальный репозиторий на вашем компьютере, удалённый — на сервере (GitHub) для совместной работы и синхронизации.
3. `Pull` скачивает изменения с удалённого репозитория и сразу вливает их в вашу текущую ветку.
4. `Push` отправляет ваши локальные коммиты на удалённый репозиторий, чтобы их увидела команда.
5. `Fetch` только скачивает изменения без слияния, а `Pull` скачивает и сразу объединяет их с вашей веткой.
6. Ветка — отдельная линия разработки внутри репозитория для параллельной работы над разными задачами.
7. В `main` неудобно работать напрямую, потому что там основная версия проекта и легко сломать код или вызвать конфликты.
8. Pull Request — запрос на слияние вашей ветки в основную с обсуждением и проверкой изменений.
9. Проверка PR нужна, чтобы другой участник нашёл ошибки и согласовал изменения до попадания в `main`.
10. Merge conflict — ситуация, когда Git не может автоматически объединить разные изменения в одном месте файла.
11. Конфликт возникает, когда два человека изменили одни и те же строки в одном файле по-разному.
12. При конфликте оставляют вариант по смыслу задачи или объединяют оба, если оба нужны.
13. Если забыть `pull`, вы работаете со старой версией и `push` может быть отклонён до обновления и слияния.
14. Маленькие понятные коммиты проще читать, проверять, откатывать и понимать историю проекта.
15. Самым сложным было разрешить merge conflict, сделать `pull` перед `push` и настроить доступ к репозиторию.
